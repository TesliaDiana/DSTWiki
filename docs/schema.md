# Звіт про структуру бази даних вікіпедії

## Таблиця: biome

**Призначення:** Зберігає інформацію про біоми ігрового світу.

| Стовпець       | Тип       | Обмеження       | Опис                                |
|----------------|-----------|----------------|------------------------------------|
| biome_id       | SERIAL    | PRIMARY KEY    | Унікальний ідентифікатор біому     |
| biome_name     | VARCHAR(32) | NOT NULL     | Назва біому                        |
| biome_location | ENUM      | NOT NULL       | Локація (EARTH, CAVE)              |
| spread         | Float     | NULL           | Розповсюдження                      |
| description    | String    | NULL           | Опис                                |

**Індекси:**  
- biome_name

**Зв’язки:**  
- Один-до-багатьох з biomeStructure  
- Один-до-багатьох з creatureBiome  
- Один-до-багатьох з itemsInBiome

---

## Таблиця: biomeStructure

**Призначення:** Зв’язує біоми та структури у світі.

| Стовпець       | Тип       | Обмеження       | Опис                  |
|----------------|-----------|----------------|----------------------|
| structure_id   | Int       | PK, FK         | Ідентифікатор структури |
| biome_id       | Int       | PK, FK         | Ідентифікатор біому     |

**Зв’язки:**  
- Багато-до-одного з biome  
- Багато-до-одного з structures

---

## Таблиця: creature

**Призначення:** Зберігає інформацію про істот.

| Стовпець        | Тип        | Обмеження       | Опис                               |
|-----------------|------------|----------------|-----------------------------------|
| creature_id     | SERIAL     | PRIMARY KEY    | Ідентифікатор істоти              |

**Індекси:**

**Зв’язки:**  
- Один-до-багатьох з creatureBiome

- Один-до-багатьох з creatureForSeason

## Висновок щодо структури

**Вибір схеми:** центральними сутностями є biome, creature, item, events, structures, seasons, gameCharacter. Використано проміжні таблиці для зв’язків багато-до-багатьох, самозв’язки для сезонів та структур.

**Нормалізація:** схема відповідає 3NF.  

**Компроміси:** використання ENUM та JSON для спрощення роботи з напівструктурованими даними.

**Індексація:** на ключові поля, зовнішні ключі та часто використовувані атрибути.

**Масштабованість:** схема дозволяє легко додавати нові біоми, істот, предмети та події без зміни основної структури.
